#  IT Asset & HR Manager

##  Тема та Мета проєкту
**Тема:** Консольна система управління персоналом та обліку комп'ютерної техніки в ІТ-компанії.

**Мета:** Демонстрація практичного застосування принципів Об'єктно-Орієнтованого Програмування (ООП) мовою Python. Проєкт створено як навчальний інструмент для закріплення навичок роботи з класами, 
інкапсуляцією, наслідуванням, поліморфізмом, а також для роботи з файловими системами та обробки винятків (валідації вводу).

##  Основний функціонал
Система надає інтерактивне консольне меню з наступними можливостями:
* **Управління персоналом:** Найм Розробників та Менеджерів, пошук працівників за іменем, звільнення.
* **Управління технікою:** Додавання на склад Ноутбуків та Моніторів, повне списання обладнання.
* **Бізнес-логіка:** * Видача техніки працівникам із суворим контролем лімітів (напр., макс. 1 ноутбук та 2 монітори на людину).
  * Повернення техніки від працівника на склад.
* **Аналітика та Звітність:**
  * Перегляд короткого Дашборду (загальна статистика по людях та складам).
  * Генерація розгорнутого текстового звіту (`report.txt`).
  * Експорт бази даних у формат `.csv` для подальшого аналізу в Excel.
* **Безпека та Історія:** Ведення автоматичного журналу подій (`audit.log`), де фіксуються всі важливі дії з відмітками часу.

## Спосіб реалізації та Архітектура
Проєкт побудовано за принципами модульної архітектури без використання сторонніх бібліотек (лише вбудовані можливості Python).

* **Моделі даних (ООП):** * Реалізовано базові класи `Employee` та `Equipment`, від яких наслідуються конкретні сутності (Розробник/Менеджер, Ноутбук/Монітор). 
  * Усі критичні дані інкапсульовані (приховані) і доступні лише через безпечні геттери.
  * Використано поліморфізм для специфічної поведінки різних посад (метод `do_work()`).
  * Ідентифікація об'єктів відбувається автоматично за допомогою генерації унікальних `UUID`.
* **Контролер (Бізнес-логіка):** За управління списками, перевірку лімітів та зв'язки між об'єктами відповідає єдиний клас-менеджер `CompanyManager`.
* **Збереження стану:** Дані автоматично серіалізуються та зберігаються у форматі `JSON` при виході з програми, та відновлюються при наступному запуску. Зв'язки між працівниками та їхньою технікою зберігаються коректно.
* **Користувацький інтерфейс:** Інтерактивне термінальне меню з очищенням екрану та глобальним перехопленням помилок. Реалізовано універсальні валідатори вводу, що унеможливлюють "падіння" програми через некоректні дані від користувача.

## Як запустити
1. Переконайтеся, що у вас встановлено Python 3.x.
2. Склонуйте або завантажте репозиторій.
3. Запустіть головний файл програми з кореневої папки:
